# LMS QA Pipeline

Инструмент автоматизированного QA для двуязычного учебного контента: сверяет
русскую и азербайджанскую версии урока — конфигурации уровней, переводы,
структуру, ответы, изображения — и собирает цветной side-by-side отчёт.

Оригинальная система работает на Google Apps Script и пишет отчёты в Google
Sheets; выгрузка уровней со страниц LMS делается userscript'ом для Tampermonkey.
Логика сравнения вынесена в чистый JS, поэтому её можно гонять и без Google —
на этом построено демо.

## Демо

**- [Живое демо](https://larina-ch-daria.github.io/LMS-parser/demo)**

Работает на синтетических обезличенных данных. Нажми «Выгрузить уровни» в
плашке Tampermonkey (как в проде) — данные пройдут пять стадий и превратятся в
сверку. Переключатель показывает урок с ошибками и здоровый урок.

Что видно в демо:
- **🔴 критично** — структурные расхождения (тип, баллы, число ответов/картинок)
  и ошибки перевода (не совпало число: `2 + 3 = 5` против `2 + 3 = 6`),
  а также уровень, которого нет в одной из версий;
- **🟠 проверить** — то, что скрипт не проверяет сам (ссылки, материалы);
- **🟢 совпадает** — перевод прошёл проверку (числа, плейсхолдеры `{{…}}`,
  операторы и соотношение длин на месте).

## Как это устроено

```
[страница урока в LMS]
      │  Tampermonkey парсит DOM
      ▼
[webhook · Apps Script Web App] ──► LMS_LEVELS · RAW_API
      │
      ▼
buildPairs        ──► LMS_PAIRS        пары РУ↔АЗ по pairKey = taskSeq||order
diffConfigs       ──► CONFIG_DIFFS     расхождения конфигов
parseNodes        ──► RAW_PARSED_NODES дерево узлов уровня
buildLevelQAView  ──► LEVEL_QA_<урок>  финальный отчёт
```

## Структура репозитория

```
demo/                        самодостаточное браузерное демо (GitHub Pages)
tampermonkey/                userscript выгрузки уровней из LMS
apps-script/
  webhook/                   отдельный Web App: приём данных от Tampermonkey
  personal-table/            bound-скрипт личной таблицы: меню + загрузчик
  qalib/                     библиотека: стадии 2–5 + сборка QA-отчёта
docs/                        скриншоты
```

Три Apps Script проекта отражают реальную архитектуру: тонкая личная таблица
оборачивает вызовы общей библиотеки `QALib`, а вебхук развёрнут отдельно

## Конфигурация продовой части

Секреты в репозитории не хранятся. Значения лежат в Script Properties
соответствующего проекта; в репозитории — только шаблоны:
- `apps-script/personal-table/Properties.example.gs` — `SCRIPT_URL`, `CLIENT_ID`,
  `CLIENT_SECRET` и т.д. (OAuth к API и адрес вебхука).
- `apps-script/qalib/Properties.example.gs` — `QA_RESULTS_ROOT_FOLDER_ID`
  (общая корневая папка Диска для отчётов).

Скопируй нужный шаблон в `Properties.gs` (он в `.gitignore`) и подставь свои
значения либо запусти `setupConfig()` / `setupQalibConfig()`.
